# dHash布局指纹算法 - 测试报告

## 算法概述

通过将页面控件树转化为"布局骨架图"，再对骨架图计算dHash，生成页面唯一性指纹。动态内容（图片、视频、文字）被替换为固定色块，只保留组件的位置和大小信息。

**指纹公式：**
```
fingerprint = Activity名 + dHash(骨架图)
```

**判定规则：**
- Activity名不同 → 直接判为不同页面（不比较hash）
- Activity名相同 → 比较dHash汉明距离，距离 < 12 为同模板

## 测试结果

### 测试1：同一页面稳定性

**操作：** 在B站首页连续dump控件树5次，计算指纹

**结果：** 5次指纹完全相同 ✓

```
第1次: ...00800080000000000000
第2次: ...00800080000000000000
第3次: ...00800080000000000000
第4次: ...00800080000000000000
第5次: ...00800080000000000000
```

**结论：** 同一页面不操作时，指纹100%稳定。

---

### 测试2：滚动后指纹稳定性

**操作：** 在首页向下滚动一屏后重新计算指纹

**结果：** 汉明距离 = 16

```
滚动前: ...00800080000000000000
滚动后: ...00000000000000800000
```

**结论：** 滚动后指纹有变化，因为顶部搜索栏收缩、频道Tab隐藏导致非列表区域组件位置变化。

**对遍历的影响：** 无。遍历代码中只在"Activity跳转后进入新页面"时计算指纹，不会在同一页面滚动后重新计算。

---

### 测试3：不同页面指纹差异

**操作：** 从首页跳转到搜索页，比较两个页面指纹

**结果：** 不同页面 ✓

```
首页 (.MainActivityV2):                         ...00800080000000000000
搜索页 (BiliMainSearchActivity):                 ...00000000000000000000
汉明距离: 17
判定: Activity名不同 → 直接判为不同页面
```

**结论：** Activity名不同时，无需比较hash即可快速判定为不同页面。

---

### 测试4：同模板不同内容（核心测试）

**操作：** 从首页分别点击两个不同的视频，进入视频详情页，比较指纹

**结果：** 同模板 ✓，汉明距离 = 3

```
视频A详情页: 组件数=183, hash=...00000000040202010800
视频B详情页: 组件数=176, hash=...00000000040202010440
同一Activity: UnitedBizDetailsActivity
汉明距离: 3 (< 阈值12)
判定: 同模板，不会重复截图
```

**骨架图对比：**
```
视频A                    视频B
────────────────        ────────────────
──··············        ──··············
──··············        ──··············
──··············        ──··············
──··············        ──··············
────────────────        ────────────────
────────────────        ────────────────
────────────────        ────────────────
────────········        ────────········
────────────────        ────────────────
────··────····──        ────··────····──
──────··········        ──────··········
──────··────────        ──────··────────
──············──        ──············──
──··············        ──··············
──······────────        ──······────────
```

**结论：** 两个不同视频的详情页，内容完全不同（不同标题、不同评论、不同UP主），但布局骨架几乎一样，指纹只差3bit。遍历时只需截图第一个视频详情页，后续所有视频详情页自动跳过。

## 性能数据

| 指标 | 数值 |
|------|------|
| 单次指纹生成 | 0.13 ms |
| 单次相似度查找（100个已知指纹） | 0.013 ms |
| 骨架图尺寸 | 64 × 128 像素 |
| dHash长度 | 128 bit (32字符hex) |

## 对遍历效率的提升

| 场景 | 旧方案（前2层TypeName） | 新方案（dHash） |
|------|----------------------|---------------|
| 两个视频详情页 | 可能判为不同（子节点数变化） | 判为相同（距离3）✓ |
| 首页滚动后 | 指纹不变 | 指纹变化但不影响（遍历中不会触发） |
| 同Activity不同功能页 | 粗粒度无法区分 | 可通过距离区分（如距离>12则不同） |
| 计算开销 | 极低 | 极低（0.13ms/次） |
